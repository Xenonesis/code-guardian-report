# Python Taint Analysis - Quick Start Guide\n\n## Overview\n\nThe code-guardian-report project includes an enterprise-grade Python taint/data-flow analyzer that detects user-controlled data flowing into dangerous operations.\n\n## Running the Analyzer\n\n### 1. Standalone Test\n\nQuick smoke test to verify the parser works:\n\n`bash\nnpx tsx test-python-parser.ts\n`\n\nExpected output:\n\n`\n==============================\n PYTHON DATA FLOW TEST START \n==============================\n\nInitializing parser...\nParser initialized successfully\n\nRunning analysis...\n\n==================================\n ANALYSIS RESULT\n==================================\n\nIssues found: 1\n[\n  {\n    \"id\": \"...\",\n    \"type\": \"Python Data Flow\",\n    \"category\": \"Injection\",\n    \"message\": \"Tainted data reaches os.system\",\n    \"severity\": \"Critical\",\n    ...\n  }\n]\n\n==============================\n TEST FINISHED\n==============================\n`\n\n### 2. Integration Tests\n\nRun the full test suite including Python analyzer tests:\n\n`bash\nnpm test -- --testPathPattern=pythonAnalyzerIntegration\n`\n\n### 3. Accuracy Tests\n\nRun broader accuracy tests that include Python scenarios:\n\n`bash\nnpm test -- --testPathPattern=analysisAccuracyTest\n`\n\n### 4. Programmatic Usage\n\n#### Via EnhancedAnalysisEngine (recommended)\n\n``typescript\nimport { EnhancedAnalysisEngine } from \"./src/services/enhancedAnalysisEngine\";\nimport JSZip from \"jszip\";\nimport fs from \"fs\";\n\nconst zip = new JSZip();\n// Add files to zip...\nconst blob = await zip.generateAsync({ type: \"blob\" });\n\nconst engine = new EnhancedAnalysisEngine();\nconst results = await engine.analyzeCodebase(blob);\n\nconsole.log(\"Found\", results.issues.length, \"issues\");\nresults.issues.forEach(issue => {\n  if (issue.type === \"Python Data Flow\") {\n    console.log(`[${issue.severity}] ${issue.message} at line ${issue.line}`);\n  }\n});\n``\n\n#### Via PythonDataFlowAnalyzer Directly\n\n``typescript\nimport { PythonDataFlowAnalyzer } from \"./src/services/analysis/PythonDataFlowAnalyzer\";\n\nconst analyzer = new PythonDataFlowAnalyzer();\nawait analyzer.init();\n\nconst issues = analyzer.analyzeDataFlow([\n  {\n    filename: \"app.py\",\n    content: `\nimport os\nuser_input = input()\nos.system(user_input)  # VULNERABLE: command injection\n    `,\n  },\n]);\n\nconsole.log(issues);\n``\n\n#### With Custom Configuration\n\n`typescript\nconst analyzer = new PythonDataFlowAnalyzer({\n  sources: [\n    \"input\",\n    \"sys.argv\",\n    \"request.args\",\n    \"my_custom_input_func\",  // Add custom source\n  ],\n  sinks: [\n    \"os.system\",\n    \"subprocess.run\",\n    \"my_dangerous_func\",  // Add custom sink\n  ],\n  sanitizers: [\n    \"escape\",\n    \"html.escape\",\n    \"my_custom_sanitizer\",  // Add custom sanitizer\n  ],\n});\n\nawait analyzer.init();\nconst issues = analyzer.analyzeDataFlow(files);\n`\n\n## Detected Vulnerabilities\n\n### Sources (Entry Points)\n\n- `input()` – user text input\n- `sys.argv[n]` – command-line arguments\n- `os.getenv()` – environment variables\n- `request.args`, `request.form`, `request.cookies` – Flask HTTP request data\n- `request.json`, `request.data` – Flask request body\n- Flask `request` object\n- Django `HttpRequest` object\n- Werkzeug request objects\n\n### Sinks (Dangerous Operations)\n\n**Command Injection** (Critical)\n- `os.system()`\n- `subprocess.run()`, `subprocess.call()`, `subprocess.spawn()`\n\n**Code Injection** (Critical)\n- `eval()`\n- `exec()`\n\n**Unsafe Deserialization** (High)\n- `pickle.loads()`, `pickle.load()`\n- `yaml.load()` (without `Loader=yaml.SafeLoader`)\n\n**HTTP Requests** (Medium)\n- `requests.get()`, `requests.post()`, etc.\n- `urllib.request.urlopen()`\n\n### Sanitizers (Data Cleaning Functions)\n\n- `urllib.parse.quote()` – URL encoding\n- `html.escape()` – HTML entity encoding\n- `escape()` – generic escape\n- `bleach.clean()` – HTML sanitization\n- `re.sub()` – regex-based substitution\n- `sanitize()` – convention-based sanitizer\n\n## Example Vulnerabilities Detected\n\n### 1. Command Injection\n\n`python\nimport os\n\n# VULNERABLE\nuser_cmd = input(\"Enter a command: \")\nos.system(user_cmd)  # ❌ Detected\n\n# FIXED\nimport subprocess\nuser_cmd = input(\"Enter a command: \")\nsubprocess.run([\"echo\", user_cmd], check=True)  # ✅ Parameters quarantined\n`\n\n### 2. Code Injection via eval()\n\n`python\n# VULNERABLE\ncode = input(\"Enter Python code: \")\neval(code)  # ❌ Detected\n\n# FIXED\nimport ast\ncode = input(\"Enter a literal: \")\nast.literal_eval(code)  # ✅ Safe for literals\n`\n\n### 3. Unsafe Deserialization\n\n`python\nimport pickle\n\n# VULNERABLE\ndata = request.get_data()\nobj = pickle.loads(data)  # ❌ Detected\n\n# FIXED\nimport json\ndata = request.get_data()\nobj = json.loads(data)  # ✅ JSON is safe\n`\n\n### 4. Taint Propagation Through Functions\n\n`python\ndef execute(cmd):\n    import os\n    os.system(cmd)  # ❌ Detected (cmd is tainted from input)\n\nuser_input = input()\nexecute(user_input)  # Taint flows through function call\n`\n\n### 5. Sanitization Breaks Taint\n\n`python\nimport os\nfrom html import escape\n\nuser_input = input()\nclean = escape(user_input)  # Sanitizer breaks taint chain\nos.system(clean)  # ✅ No issue (clean data)\n`\n\n## Issue Fields\n\nEach detected issue includes:\n\n`json\n{\n  \"id\": \"unique-issue-id\",\n  \"type\": \"Python Data Flow\",\n  \"category\": \"Injection\",\n  \"message\": \"Tainted data reaches os.system\",\n  \"severity\": \"Critical\",\n  \"confidence\": 95,\n  \"filename\": \"app.py\",\n  \"line\": 8,\n  \"column\": 0,\n  \"codeSnippet\": \"\",\n  \"recommendation\": \"Sanitize input\",\n  \"remediation\": {\n    \"description\": \"Sanitize\",\n    \"effort\": \"Medium\",\n    \"priority\": 5\n  },\n  \"riskRating\": \"Critical\",\n  \"impact\": \"\",\n  \"likelihood\": \"High\",\n  \"references\": [],\n  \"tags\": [\"python\"],\n  \"tool\": \"Python Analyzer\",\n  \"cvssScore\": 9,\n  \"cweId\": \"CWE-78\",\n  \"owaspCategory\": \"A03\"\n}\n`\n\n## Performance\n\n- **Time**: ~50-100ms for typical 1000-LOC Python file\n- **Memory**: Minimal (single-pass analysis)\n- **Scalability**: Can analyze entire repositories in seconds\n\n## Known Limitations\n\n1. **Single-File Analysis**: Does not track taint across imported modules.\n2. **No Complex Containers**: Limited propagation through dicts/lists/tuples.\n3. **Basic CFG**: Does not model conditional branches in detail.\n4. **Heuristic Patterns**: Some advanced Python idioms may not be recognized.\n\nThese are planned for future enhancements.\n\n## Architecture Overview\n\n`\nEnhancedAnalysisEngine\n  ├─ Phase 1: Regex-based rules (SecurityAnalyzer)\n  ├─ Phase 2: AST analysis (ASTAnalyzer for JS/TS)\n  ├─ Phase 3a: Data flow for JS/TS (DataFlowAnalyzer)\n  ├─ Phase 3b: Python taint analysis ⬅️ THIS COMPONENT\n  │    └─ PythonDataFlowAnalyzer\n  │       ├─ Parses Python with tree-sitter\n  │       ├─ Registers functions\n  │       ├─ Detects sources from configured list\n  │       ├─ Propagates taint through assignments/calls\n  │       ├─ Recognizes sanitizers and breaks flows\n  │       ├─ Detects sinks from configured list\n  │       └─ Generates SecurityIssue objects\n  ├─ Phase 4: Dependency scanning\n  └─ Output: AnalysisResults object\n`\n\n## Integration Points\n\n### With ZIP/Repository Analysis\n\nWhen you upload a ZIP file or GitHub repository with `.py` files:\n\n1. `EnhancedAnalysisEngine.analyzeCodebase()` is called\n2. Files are extracted and passed to various analyzers\n3. Python files trigger `PythonDataFlowAnalyzer.analyzeDataFlow()`\n4. Results are merged with JS/TS/other language findings\n5. Combined report is returned to UI\n\n### With File Analysis Service\n\nWhen analyzing individual files via `EnhancedFileAnalysisService`:\n\n1. `.py` files are detected\n2. Python analyzer is lazily initialized\n3. Taint issues are added to `securityFindings`\n4. Results are merged with other analysis layers\n\n## Troubleshooting\n\n### \"Parser initialization failed\"\n\n**Cause**: `tree-sitter` WASM module not available or `./public/tree-sitter-python.wasm` not found.\n\n**Solution**: Check that the WASM file exists in the project root:\n\n`bash\nls ./public/tree-sitter-python.wasm\n`\n\nIf missing, rebuild or re-initialize the project.\n\n### \"No issues detected\" (when expecting some)\n\n**Cause**: The specific pattern may not match due to:\n- Variable name not recognized\n- Source/sink not in default config\n- Code structure differs from expected\n\n**Solution**: Add custom sources/sinks or check analyzer logs:\n\n`typescript\nanalyzer.setConfig({ sources: [...], sinks: [...] });\n`\n\n### Performance degradation on large files\n\n**Cause**: Very large files (10MB+) cause slowdown.\n\n**Solution**: Consider skipping analysis for such files or optimizing the\nanalyzer's tree-walk algorithm.\n\n## Next Steps\n\n1. **Review** [PYTHON_TAINT_DESIGN.md](./PYTHON_TAINT_DESIGN.md) for detailed architecture\n2. **Run** tests to verify functionality in your environment\n3. **Configure** custom sources/sinks as needed for your codebase\n4. **Integrate** into your CI/CD pipeline via the engine API\n5. **Monitor** false positive/negative rates and tune configuration\n\n## Support & Contribution\n\nFor issues, feature requests, or contributions, please refer to the project's\nCONTRIBUTING.md and open a discussion or pull request.\n\n---\n\n*Python Taint Analyzer - Enterprise Grade Security Analysis*\n
