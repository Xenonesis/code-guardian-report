# 🚀 GitHub Analysis - Quick Reference Card

## 📋 Supported URL Formats

| Format | Example | Notes |
|--------|---------|-------|
| **Basic** | `https://github.com/owner/repo` | Analyzes default branch |
| **Branch** | `https://github.com/owner/repo/tree/branch` | Analyzes specific branch |
| **With .git** | `https://github.com/owner/repo.git` | Also supported |

## ⚡ Quick Commands

```bash
# Example URLs to try:
https://github.com/facebook/react
https://github.com/microsoft/vscode
https://github.com/yourusername/yourproject
```

## 🎯 What Gets Analyzed

### ✅ Included
- JavaScript/TypeScript (.js, .jsx, .ts, .tsx)
- Python (.py)
- Java (.java)
- C/C++ (.c, .cpp, .h)
- Go (.go)
- Rust (.rs)
- Ruby (.rb)
- PHP (.php)
- And 20+ more languages!

### ❌ Excluded
- `node_modules/`
- `.git/`
- `dist/`, `build/`
- `coverage/`
- `vendor/`
- `__pycache__/`

## 🔄 Process Flow

```
1. Paste URL → 2. Validate → 3. Preview Info → 4. Click Analyze → 5. View Results
```

## ⏱️ Expected Times

| Repository Size | Approximate Time |
|----------------|------------------|
| Small (< 50 files) | 10-30 seconds |
| Medium (50-200 files) | 30-90 seconds |
| Large (> 200 files) | 1-3 minutes |

## 🆘 Common Issues & Solutions

| Problem | Solution |
|---------|----------|
| "Repository not found" | Check URL, ensure repo is public |
| "No code files found" | Verify repo contains actual code |
| "Rate limit exceeded" | Wait 1 hour, try during off-peak |
| Download stuck | Check internet, be patient |

- ✅ Processed in your browser
- ✅ Same security as file upload

## 💡 Pro Tips

1. **Start Small**: Test with smaller repos first
2. **Check Info**: Review file count before analyzing
3. **Use Branches**: Specify branch for accurate analysis
4. **Be Patient**: Large repos take time
5. **Off-Peak**: Better rates during off-peak hours
